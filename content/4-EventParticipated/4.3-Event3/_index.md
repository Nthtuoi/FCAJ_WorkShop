# Event Recap – July 11

### Event Objectives

- Explore and stay updated on the latest security solutions and tools within the AWS cloud ecosystem.
- Enhance understanding of Service Level Agreements (SLAs), risk management mechanisms, operational monitoring, and automated alerting systems.
- Clarify the exam structure, key knowledge areas, and optimal preparation strategies for the AWS Cloud Practitioner certification.
- Bridge the gap between academic theory and real-world infrastructure deployment scenarios.
- Define a learning and career development roadmap aligned with current technology trends.

### Speaker List
- **Thinh Nguyen** 
- **Nguyễn Huỳnh Sơn** 
- **Ngô Lê Tuấn Huy** 

### Key Highlights
- Speaker 1: Optimizing Web Application Security via AWS Security Agent
* Identifying common security vulnerabilities and preventive solutions using "Edge Agents."
* Source code review and security assessment processes integrated with automated penetration testing tools.
* Analysis of security investment costs and core technical limitations.
- Speaker 2: From Service Level Agreements (SLAs) to System Monitoring
* The nature of SLA metrics and the close relationship between monitoring activities and risk management strategies.
* Real-world case study: Understanding the fundamental differences between two system endpoints.
* Methods for converting metric data streams into timely incident alerts.
- Speaker 3: Roadmap to Achieving the AWS Cloud Practitioner Certification
* Overview of the exam format, assessment objectives, and the structure of the official exam sections.
* Deep dive into four foundational knowledge areas:
* Part 1: Cloud computing principles and models.
* Part 2: Information security, data protection, and compliance standards.
* Part 3: AWS service systems and technology solutions.
* Part 4: Pricing models, budget management, and technical support plans.
* Practical insights: Time allocation strategies, official study resources, and tips for achieving high scores on multiple-choice exams.

### Key Takeaways
- The session provided me with a deeper understanding of the importance of information security right from the software development stage. I also learned how to utilize automation tools to proactively scan for vulnerabilities and minimize system security risks.
- I gained practical insights into SLA operations and alerting systems. Monitoring metrics must align closely with core processes, prioritize the actual user experience, and ensure rapid response times during incidents.
- Additionally, I developed a comprehensive understanding of the AWS Cloud Practitioner certification. Achieving success requires a study plan that effectively integrates theoretical reinforcement, hands-on lab configuration, and skill practice using mock exams.

#### Practical Application Strategy

- The knowledge gained will serve as a foundation for optimizing secure source code management, establishing strict access control policies, configuring monitoring systems, and developing automated response protocols for anomaly detection.
- A thorough understanding of these services enables the selection of appropriate solutions for specific challenges and effective resource cost management, thereby enhancing professional expertise and preparing for roles in the Cloud and DevOps fields.

### Event Experience

- This was an incredibly valuable, highly practical, and concise workshop. The technical content was conveyed in a visual and coherent manner, making it very accessible to both students and professionals just starting out in the field of AWS cloud computing.
- The event not only provided great motivation for me to continue honing my skills in Cloud/DevOps but also offered clear direction, giving me the confidence to map out my career path for the journey ahead.
